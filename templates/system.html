System

How the monitoring pipeline works from camera to alert.

This page explains the working system in a cleaner, dedicated layout.

System Workflow
01

Camera Input

Capture from laptop webcam or external IP camera source.

02

Face Mesh

Detect the face and extract the landmarks required for analysis.

03

Signal Analysis

Compute EAR, MAR, and head pose values on each frame.

04

Decision Logic

Classify the state as active, drowsy, yawning, or distracted.

05

Alert Layer

Play alarm audio and optionally prepare WhatsApp escalation.

06

Live Dashboard

Render the processed feed and live metrics to the browser.

Current Workflow

Camera -> Face Mesh -> EAR / MAR / Head Pose -> Decision Logic -> Alarm -> Live Monitoring

The browser layer now acts as the presentation and control shell while reusing the existing Python detection engine.

Design Direction

Closer to a real dashboard product

Separate pages, stronger branding, animated interface elements, and a dedicated monitor route make the project feel more production-style.
